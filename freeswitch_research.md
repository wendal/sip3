
FreeSWITCH 调研结论

对比 FreeSWITCH 的完整功能集，SIP3 在当前架构（轻量代理）下最值得优先实现的改进：

┌────────┬─────────────────────────────┬────────────────────────┐
│ 优先级 │ 功能                        │ 理由                   │
├────────┼─────────────────────────────┼────────────────────────┤
│ 🔴 高  │ SRTP + SIP/TLS              │ 生产环境安全必须       │
├────────┼─────────────────────────────┼────────────────────────┤
│ 🔴 高  │ IP ACL（CIDR 黑白名单）     │ 防扫描防滥用，代码量小 │
├────────┼─────────────────────────────┼────────────────────────┤
│ 🟡 中  │ 呼叫转移（SIP REFER）       │ 最常见的办公室通话场景 │
├────────┼─────────────────────────────┼────────────────────────┤
│ 🟡 中  │ SIP Presence / BLF          │ IP 话机状态灯需要      │
├────────┼─────────────────────────────┼────────────────────────┤
│ 🟡 中  │ SIP/WSS（WebRTC 接入）      │ 浏览器直接拨打         │
└────────┴─────────────────────────────┴────────────────────────┘

更高级功能（转码、会议、IVR、录音）需要将架构从"代理"改为"B2BUA"，是长期目标。
